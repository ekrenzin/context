# Weekly Report Skill

## When to Use

Use this skill when you need to:
- Generate automated weekly team progress summaries
- Prepare management reports with key metrics
- Track team performance trends over time
- Create visual dashboards for stakeholder reviews
- Analyze sprint completion rates and velocity
- Monitor resource utilization and productivity

## Workflow

1. **Configure Report Parameters**
   - Select team members and projects to include
   - Define date range for the reporting period
   - Choose metrics and KPIs to track
   - Set visualization preferences

2. **Aggregate Data Sources**
   - Collect task completion data from project management tools
   - Gather time tracking information
   - Pull code commit and deployment statistics
   - Retrieve meeting and collaboration metrics

3. **Process and Analyze Data**
   - Calculate completion rates and velocity metrics
   - Identify trends and performance patterns
   - Generate comparative analysis with previous periods
   - Flag potential issues or bottlenecks

4. **Generate Visual Summary**
   - Create charts for key performance indicators
   - Build progress tracking visualizations
   - Generate team productivity heatmaps
   - Compile executive summary dashboard

5. **Review and Validate**
   - Verify data accuracy and completeness
   - Check metric calculations and trends
   - Ensure visualizations are clear and actionable
   - Validate summary insights and recommendations

6. **Distribute Report**
   - Export report in required formats
   - Schedule automated delivery to stakeholders
   - Archive report for historical tracking
   - Set up alerts for significant changes

## Validation

- All configured data sources are accessible and current
- Date ranges align with team sprint cycles
- Calculated metrics match expected formulas
- Visualizations accurately represent underlying data
- Summary insights are supported by the data
- Report format meets stakeholder requirements
- Historical data is properly archived for trend analysis
